Authentication

Acme Cloud uses two authentication schemes: bearer tokens for personal access, and OAuth2 client credentials for server-to-server. All API requests must be made over HTTPS.

Bearer tokens

Generate a personal access token at acmecloud.example/account/tokens. Include it on every request:

curl -H "Authorization: Bearer acme_pat_..." https://api.acmecloud.example/v1/projects

Tokens are scoped (read / write / admin) and expire after 90 days unless you set "never".

OAuth2 client credentials

For server-to-server workloads:

curl -X POST https://auth.acmecloud.example/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=acme_cli_..." \
  -d "client_secret=..." \
  -d "scope=projects:read deployments:write"

The response contains an access_token valid for 1 hour.

Scopes

ScopePermits
projects:readList + read projects
projects:writeCreate + update projects
deployments:readList + read deployments
deployments:writeTrigger deployments
adminAll of the above + billing + member management

Token rotation

We rotate API keys quarterly. You'll receive an email 30 days before any token expires.
