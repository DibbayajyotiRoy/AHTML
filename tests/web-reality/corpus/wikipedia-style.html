Distributed hash table

A distributed hash table (DHT) is a class of decentralized distributed system that provides a lookup service similar to a hash table: (key, value) pairs are stored in a DHT, and any participating node can efficiently retrieve the value associated with a given key.

History

DHT research was originally motivated by peer-to-peer (P2P) systems such as Freenet, Gnutella, BitTorrent, and Napster. Most early systems used central indices, which made them vulnerable to legal and technical attack.

Properties

DHTs typically emphasize the following properties:

  • Autonomy and decentralization: nodes collectively form the system without any central coordination.
  • Fault tolerance: the system is reliable even with nodes continuously joining, leaving, and failing.
  • Scalability: the system functions efficiently with thousands or millions of nodes.

Structure

Keyspace partitioning

Most DHTs use some variant of consistent hashing or rendezvous hashing to map keys to nodes. The two algorithms have similar properties but rendezvous hashing handles node failures more gracefully.

Overlay network

Nodes maintain a set of links to other nodes (the overlay network). Together, these links form a network that allows the routing of messages to the node responsible for a given key.

Examples

  • Chord (MIT, 2001)
  • Kademlia (NYU, 2002) — used by BitTorrent's mainline DHT and IPFS
  • Pastry (Microsoft Research, 2001)
  • Tapestry (UC Berkeley, 2001)
