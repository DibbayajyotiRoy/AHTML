How I cut our build pipeline from 14 minutes to 90 seconds

By Alex Chen · May 10, 2026 · 8 min read

Our continuous integration was the slowest part of shipping. Every PR took 14 minutes to go from push to "ready to merge." Across 60+ engineers, that was burning a day a week in waiting time.

Diagnosis

I broke the pipeline into measurable stages. node_modules install was 6m20s. Test execution was 4m. Docker build was 3m. Linting was 30s.

1. Cache node_modules properly

Cache hit rate went from 30% to 94% after we keyed the cache on the package-lock.json hash only (not the branch).

2. Parallelize tests

Sharded the test suite into 4 parallel jobs. Each job runs ~25% of the tests. Total wall-clock for the test stage dropped from 4m to 1m10s.

3. Pre-warm the Docker layer cache

We pre-built and pushed a base layer with all our system deps as a nightly. PR builds reuse this layer.

Results

14m → 90s. Engineer wait-time per week: -22%. Mean time to deploy: -41%.
